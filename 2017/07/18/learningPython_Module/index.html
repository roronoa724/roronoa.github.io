一步一步学Python--模块

发表于 2017-07-18 | 分类于 Python

前言

模块(Module)是Python程序的最高层组织单元。可以说，我们在Python世界中书写的代码都是必然存在于模块之中的。更具体而言，每一个文件就是一个模块，且模块可以引用(import)其他的模块。通常情况下，Python程序就是众多错综复杂且相互依赖的模块所组成的。所以了解模块对于我们了解Python项目的整体架构和运作方式都有着重要的意义。

为什么需要模块化

一言蔽之：模块化是一种处理复杂系统分解为更好的可管理模块的方式。
详细来说，模块化能够带来许多好处，下面列举三点：

  • 代码重用
    因为模块本质上就是一个文件，与交互命令行中输入的代码不同，它可以被永久保存，并能作为一个重用单元按需重新装载或运行。所以使用模块可以用来实现一些公共服务供系统其他模块调用。
  • 分隔系统命名空间
    模块还可以看做是对象名称，也称为模块的属性(attributes)的集合，在Python中，每个模块都具有自己独立的命名空间(namespace)，相同名字的函数和变量完全可以分别存在不同的模块中，最大程度避免了函数名和变量名冲突。
  • 大大提高了代码的可维护性和开发效率。

目前为止还比较抽象，现在我们来对Python程序的整体结构进行一个概览，以来更好的理解模块在Python程序中充当的角色。

模块是如何运作的？

在理解模块的运作方式之前，先让我们来看一下通常的Python程序结构是怎样的。

Python程序概览

先把自己平时写着玩儿的脚本抛到脑后，大部分生产环境的Python项目都是多文件(multifile,也就是多模块)系统。通常来说，这些文件会包含有Python声明语句(statements)，并且至少包含一个顶层(top-level)的主(main)文件，以及0个或多个补充文件，也就是我们所说的模块。
主文件的作用在于控制程序的核心执行流程，模块则负责为主文件和其他模块提供服务。如何使用其他模块提供的服务就引出了Python模块知识体系内最重要的关键字：import。

为了实现这一目标，Python会顺次执行被import文件当中的声明语句。需要注意的有以下几点：

  • 在Python中，除非显示的在运行时(runtime)使用import关键字声明引用，否则是不会出现跨文件的模块链接(linking)关系解析的。亦即一个文件如果没有被import过的话，则是不会被解析的(主文件除外)。
  • import声明中指定的模块名称在这里起到两个作用，其一为标识需要加载的模块文件名称(即b.py的文件名称)，其二该名称会作为装载模块的变量名(variable)供后续使用。
  • 模块中定义的对象也是在import后，在运行时生成的。举例来说，在主文件a引用模块b之后，可以通过b.spam引用模块b中的属性。这里的spam既可以是一个变量，也可以是一个函数。这种object.attribute类型的调用方式是引用模块当中变量的最常见方式。
